PART 1: ENVIRONMENT SETUP
─────────────────────────────────────────────────────────────────────────────

Requirement Check:
  - Node.js 18 or higher
  - npm, yarn, or pnpm installed
  - Modern web browser (Chrome recommended)
  - ~500MB disk space for node_modules

Verify Installation:
  node --version        # Should show v18.x or higher
  pnpm --version        # Should show 9.x or higher
  npm --version         # Alternative to pnpm

PART 2: PROJECT INSTALLATION
─────────────────────────────────────────────────────────────────────────────

Step 1: Navigate to Project
  cd /vercel/share/v0-project

  Verify location:
  pwd                   # Should end with /v0-project
  ls package.json       # Should exist

Step 2: Install Dependencies
  pnpm install

  Expected output:
  ✓ Installed 234 packages in 45s
  (number may vary)

  What gets installed:
  - Next.js 16 (React framework)
  - React 19 (UI library)
  - TailwindCSS (styling)
  - shadcn/ui (components)
  - PeerJS (multiplayer)
  - qrcode.react (QR generation)
  - Web Audio API (built-in)

Step 3: Verify Installation
  ls node_modules       # Should have many folders
  ls package-lock.json  # or pnpm-lock.yaml
  pnpm list next        # Show Next.js version

PART 3: STARTING THE GAME
─────────────────────────────────────────────────────────────────────────────

Start Development Server:
  pnpm dev

  Expected output:
  ▲ Next.js 16.0.0
  - Local:        http://localhost:3000
  - Network:      http://192.168.x.x:3000
  ✓ Ready in 435ms

Wait for "Ready" message before opening browser!

PART 4: ACCESSING THE GAME
─────────────────────────────────────────────────────────────────────────────

Option A: Automatic
  Browser should open automatically to http://localhost:3000

Option B: Manual
  1. Open browser (Chrome recommended)
  2. Go to address bar
  3. Type: http://localhost:3000
  4. Press Enter

  You should see:
  - Title: "CYBERLANE"
  - Main menu with buttons
  - "PLAY SINGLE PLAYER" button
  - "PLAY MULTIPLAYER" button

PART 5: PLAYING SINGLE-PLAYER
─────────────────────────────────────────────────────────────────────────────

Step 1: Start Game
  - Click "PLAY SINGLE PLAYER" button
  - Main menu disappears
  - Game screen appears

Step 2: Configure Controls (Choose One Method)
  Use KEYBOARD:
    - Arrow Keys: ↑↓←→
    - Or WASD keys
    - Hold down to move continuously

  Use MOUSE:
    - Move cursor around the game canvas
    - Player character automatically moves toward cursor
    - No click needed

  Use MOBILE BUTTONS:
    - Look bottom-right corner of screen
    - Four arrow buttons (▲ ◄ ▼ ►)
    - Tap to move in that direction

Step 3: Start Playing
  - Click "START GAME" button
  - Canvas shows black background with cyan player circle
  - Red obstacles fall from top after 1-2 seconds
  - Player starts in center-bottom of canvas

Step 4: Play the Game
  - GOAL: Dodge falling red obstacles
  - COLLECT: Glowing powerup orbs
    * Blue shield: Block 1 hit
    * Orange speed: Move 1.5x faster
    * Blue slow: Move slower (tactical)
  - SURVIVE: As long as possible
  - Score increases every frame survived

Step 5: Game Over
  - Touching obstacle = Game Over (unless shielded)
  - Screen shows final score and waves completed
  - Button options:
    * "PLAY AGAIN" - restart game
    * "MAIN MENU" - return to menu

PART 6: PLAYING MULTIPLAYER
─────────────────────────────────────────────────────────────────────────────

HOST SETUP (Computer A):
  1. Click "PLAY MULTIPLAYER" button
  2. Click "CREATE GAME" button
  3. Game code appears (e.g., "HOST-ABC123")
  4. QR code displays for easy sharing
  
  Copy game code or show QR code to guests:
    - Game code: Text form for manual entry
    - QR code: Scan with phone camera
  
  5. Wait for guests to join (see player count increase)
  6. Once ready, click "START GAME"

GUEST SETUP (Computer B / Phone):
  1. Click "PLAY MULTIPLAYER" button
  2. Click "Join Game" button
  3. Enter host's game code when prompted
     OR scan the QR code from host
  4. Click "Join" or "Connect"
  5. Wait for "Connected" status
  6. Host starts game when ready

During Multiplayer Game:
  - All players dodge same obstacles
  - Highest score wins
  - Real-time synchronization
  - Player positions visible on canvas
  - Powerups are personal (only you get them)

PART 7: STOPPING THE SERVER
─────────────────────────────────────────────────────────────────────────────

When done playing:
  - In terminal where `pnpm dev` is running
  - Press: Ctrl+C
  - Wait for prompt
  - Type: y (yes to stop)
  
  Or force stop:
  - Press: Ctrl+C twice

Verify stopped:
  - Terminal shows command prompt ($ or >)
  - No more "Ready" messages

================================================================================
COMMON QUESTIONS & ANSWERS
================================================================================

Q: Do I need internet?
A: Only for initial npm package download. Game runs fully offline after install.

Q: Can I play on phone?
A: Yes! Two ways:
   1. Single-player: http://192.168.1.X:3000 (replace X with your IP)
   2. Multiplayer: Host on PC, join from phone

Q: How do I find my IP address?
A: When pnpm dev starts, terminal shows: "Network: http://192.168.1.X:3000"
   Use that address on phone.

Q: Is audio required?
A: No, game works without sound. Audio is optional enhancement.

Q: Do high scores save?
A: Currently saved in browser's local storage only.
   Clearing cache = lose scores.

Q: Can I deploy this?
A: Yes! 
   pnpm build
   pnpm start
   Or push to GitHub and use Vercel auto-deployment.

Q: Does multiplayer work over internet?
A: Currently requires same WiFi network due to free STUN servers.
   Could be enhanced with commercial TURN server.

Q: What if I forget my game code?
A: Create a new game - you get a new code each time.

Q: How do I play with friends far away?
A: Current: Use VPN to connect to same network, or use Hamachi/Tailscale
   Future: Add cloud-based relay server

Q: Can I customize colors/sounds?
A: Yes! Edit:
   - Colors: app/globals.css
   - Sounds: lib/audioManager.ts
   - Visuals: components/GameCanvasEnhanced.tsx
